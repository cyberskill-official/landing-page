## §1 — Description (BCP-14 normative)

1. **MUST** ship the following decoders locally under `apps/web/public/decoders/` per master plan §5.3:

   | Decoder | Files | Purpose | Approx. size |
   |---|---|---|---|
   | Draco | `draco/draco_decoder.wasm`, `draco/draco_wasm_wrapper.js` | static-mesh decode (signs, props) | ~95 KB |
   | Meshopt | `meshopt/meshopt_decoder.module.js` | rigged-mesh decode (Lumi) | ~50 KB |
   | Basis Universal | `basis/basis_transcoder.js`, `basis/basis_transcoder.wasm` | KTX2 texture transcode (all PBR textures) | ~95 KB |

2. **MUST NOT** load decoders from any CDN at runtime. Forbidden hosts (regex enforced by FR-OPS-013 CI gate):
   - `unpkg.com`
   - `cdn.jsdelivr.net`
   - `cdnjs.cloudflare.com`
   - `*.threejs.org`
   - `*.googleapis.com`
   - `gstatic.com`

3. **MUST** total decoder weight ≤ **240 KB combined** (gzipped if Vercel auto-compresses; raw bytes are larger). Master plan §5.3 ceiling.

4. **MUST** ship `scripts/sync-decoders.mjs` that updates the bundled decoders to match the versions pinned in `package.json`:
   - `three@^0.184.x` → fetch `examples/jsm/libs/draco/*` and `examples/jsm/libs/basis/*` from the matching git tag
   - `meshoptimizer@^0.21.x` → fetch `js/meshopt_decoder.module.js` from npm pack
   - Idempotent: running on unchanged versions produces no file changes (verified by SHA-256 comparison).

5. **MUST** be loaded on-demand by R3F via Drei's `useGLTF.preload` decoder-path config:
   ```ts
   useGLTF.setDecoderPath("/decoders/draco/");
   useGLTF.setMeshoptDecoder(MeshoptDecoder); // imported from local path
   useGLTF.setKTX2Loader(ktx2Loader);          // ktx2Loader configured to /decoders/basis/
   ```

6. **MUST NOT** load decoders eagerly. They MUST load only when the first .glb / .ktx2 needs them (i.e. inside `dynamic-three.ts` import boundary per FR-WEB-005).

7. **MUST** include a generated `apps/web/public/decoders/README.md` explaining:
   - Version pinning to three.js / meshoptimizer
   - How to bump (`pnpm sync-decoders`)
   - CSP justification (no CDN allowlist needed)
   - File integrity hash (SHA-256 per file, regenerated by sync script)

8. **MUST** be served with `Cache-Control: public, immutable, max-age=31536000` headers via Next.js's `headers()` config — versioned via package.json, so cache busting requires a sync-decoders run + content-hash change.

9. **MUST** be served with `Content-Type: application/wasm` for `.wasm` files (browsers require this for `WebAssembly.instantiateStreaming`).

10. **MUST** validate WASM streaming compilation works (vs synchronous `instantiate`). Adding `Cross-Origin-Embedder-Policy` + `Cross-Origin-Opener-Policy` headers if needed for SharedArrayBuffer.

11. **MUST** ship per-file SHA-256 integrity hash in the README; sync-decoders verifies on fetch (rejects if upstream tag's bytes changed unexpectedly).

12. **SHOULD** ship a `decoders.manifest.json` listing each decoder's version + hash + size, consumed by FR-OPS-001's pipeline report and FR-OPS-003's PR comment.

## §2 — Why this design

**Why local bundle (not CDN)?** Three independent reasons:
1. **CSP compliance** — FR-A11Y-001 §7.2 forbids third-party script/wasm sources for security (no `unsafe-eval`, no third-party origins). CDN-hosted decoders would force `script-src https://unpkg.com` etc., expanding the attack surface.
2. **Offline dev** — Frontend lead works on planes / coffee shops with flaky wifi. CDN dependency means Lumi won't load in dev mode without internet.
3. **Reliability** — unpkg/jsDelivr/cdnjs have measurable outage history (3-5 incidents/year). A CDN outage breaks Lumi rendering site-wide. Local bundle is uptime-coupled to Vercel only.

**Why 240 KB ceiling?** Master plan §5.3 budget. Includes 95 KB Draco + 50 KB Meshopt + 95 KB Basis. Adds ~80 KB gzipped to first-asset-needing-it payload, amortized across decoders shared between assets. Mobile cold-load over 4G: ~200 ms additional. Acceptable; tested via FR-PERF-010.

**Why pin to package.json version (not "latest")?** Decoder protocols are not strictly forward-compatible. A Three.js bump that ships a new Draco decoder is fine if the encoder (in CI Docker image) matches. Pinning prevents skew between encode-time and decode-time versions.

**Why on-demand (not preload in <head>)?** Lazy. Most users on the site never reach Scene 5 / Lumi-heavy pages. Bundling decoders into the initial HTML payload adds 80 KB to the cold-load for pages that may not use them. Drei's lazy loading is the right contract.

**Why immutable cache-control?** Decoder versions only change when developer-initiated (sync-decoders run). Once shipped, the byte content never changes for that path. 1-year immutable cache is correct.

**Why integrity hash?** Defense-in-depth. If an attacker compromised our CI and replaced `draco_decoder.wasm`, integrity hash catches it. Manual sha256 in README is checked at sync time and during CI build.

```ts
// scripts/sync-decoders.mjs (ESM Node 22+)
import { fetch } from "undici";
import { writeFile, mkdir, readFile, stat } from "node:fs/promises";
import { createHash } from "node:crypto";
import { resolve } from "node:path";

interface DecoderSource {
  name: string;
  source: "three" | "meshopt";
  files: { remote: string; local: string }[];
}

const SOURCES: DecoderSource[] = [
  {
    name: "draco",
    source: "three",
    files: [
      { remote: "examples/jsm/libs/draco/draco_decoder.wasm",     local: "draco/draco_decoder.wasm" },
      { remote: "examples/jsm/libs/draco/draco_wasm_wrapper.js",  local: "draco/draco_wasm_wrapper.js" },
    ],
  },
  {
    name: "basis",
    source: "three",
    files: [
      { remote: "examples/jsm/libs/basis/basis_transcoder.js",    local: "basis/basis_transcoder.js" },
      { remote: "examples/jsm/libs/basis/basis_transcoder.wasm",  local: "basis/basis_transcoder.wasm" },
    ],
  },
  {
    name: "meshopt",
    source: "meshopt",
    files: [
      { remote: "js/meshopt_decoder.module.js", local: "meshopt/meshopt_decoder.module.js" },
    ],
  },
];

export async function syncDecoders(opts: { threeVersion: string; meshoptVersion: string; outDir: string; dryRun?: boolean }) {
  const manifest: Record<string, { version: string; sha256: string; bytes: number; path: string }> = {};
  let totalBytes = 0;
  for (const src of SOURCES) {
    const version = src.source === "three" ? opts.threeVersion : opts.meshoptVersion;
    const baseUrl = src.source === "three"
      ? `https://raw.githubusercontent.com/mrdoob/three.js/r${version.replace(/[\^~v]/g, "")}`
      : `https://unpkg.com/meshoptimizer@${version}`;  // SYNC-TIME only; runtime never hits unpkg
    for (const file of src.files) {
      const url = `${baseUrl}/${file.remote}`;
      const res = await fetch(url);
      if (!res.ok) throw new Error(`Failed to fetch ${url}: ${res.status}`);
      const bytes = Buffer.from(await res.arrayBuffer());
      const sha = createHash("sha256").update(bytes).digest("hex");
      const localPath = resolve(opts.outDir, file.local);
      if (!opts.dryRun) {
        await mkdir(resolve(opts.outDir, file.local.split("/")[0]), { recursive: true });
        await writeFile(localPath, bytes);
      }
      manifest[file.local] = { version, sha256: sha, bytes: bytes.length, path: file.local };
      totalBytes += bytes.length;
    }
  }
  if (totalBytes > 240 * 1024) {
    throw new Error(`Decoder bundle exceeded 240 KB ceiling: ${totalBytes} bytes`);
  }
  if (!opts.dryRun) {
    await writeFile(resolve(opts.outDir, "decoders.manifest.json"), JSON.stringify(manifest, null, 2));
  }
  return { totalBytes, manifest };
}
```

## §7 — Failure modes

| Failure | Detection | Recovery |
|---|---|---|
| CDN reference creeps into code | AC#3 grep + FR-OPS-013 gate | CI blocks; offender refactors to local path |
| Decoder version skew (encode v1, decode v2) | Lumi fails to load on prod, works in dev | Pin in package.json + sync-decoders match-check; CI fails on mismatch |
| Combined size > 240 KB | AC#2 | sync-decoders throws; bump master-plan §5.3 ceiling or drop unused decoder |
| Wrong Content-Type breaks WASM streaming | Browser console: "incorrect MIME type" | next.config.ts headers() rule per .wasm |
| Cache-Control not immutable → re-fetches | Network waterfall on warm reload | Verify headers() rule scope: `/decoders/:path*` |
| WASM blocked by COEP/COOP | Browser console: "blocked by Cross-Origin-Embedder-Policy" | Add `crossorigin="anonymous"` to script tag if needed (Drei handles this) |
| sync-decoders fails to fetch (rate limit on raw.githubusercontent) | CI log 429 | Retry with exponential backoff; cache fetched bytes in CI artifact |
| Bundled `meshopt_decoder.module.js` not ES module compatible | Module load error | Verify pinned version is the `.module.js` (ESM) variant, not the UMD |
| Decoder fetched eagerly (initial page payload bloat) | AC#11 | Audit dynamic-three.ts boundary; ensure decoders inside the dynamic import |
| Integrity hash mismatch after upstream Three.js tag rebuild | Sync re-run fails | Investigate upstream change; update README hash if intentional |
| Manifest.json schema validation fails | AC#6 | Vitest unit asserts shape |
| Three.js tag URL pattern breaks on raw GitHub | Sync log 404 | Three.js bumped CDN structure; update SOURCES URLs in sync script |
| Offline dev fails (decoder paths still hit network) | Manual: disable wifi, reload | Verify next-dev serves /public statically; cache headers may force revalidation |
| Vercel gzip strips integrity (different bytes over wire vs disk) | Integrity check fails on prod | Pre-compute hash of raw bytes; Vercel decompresses before checksum |

## §9 — Notes

**On future protocol bumps:** When three.js / meshopt bump to a new decoder version, run `pnpm sync-decoders`, commit the new bytes + manifest. CI assertion ensures CSP not regressed. Document version-bump in `apps/web/public/decoders/README.md`.

**On serving from a CDN behind the same origin:** Vercel serves `/decoders/*` from their edge CDN automatically. The "no CDN" rule is about *third-party* origins; Vercel's edge counts as same-origin for CSP purposes.

**On meshopt decoder bundling alternative:** Could import from `three-stdlib` package directly (no /public/ shipping). Chose /public/ to:
1. Cache control identical for all 3 decoders.
2. Pre-fetch ability via `<link rel="preload" as="fetch" crossorigin>` if needed.
3. Inspect-in-browser-Network-tab visibility for debugging.

**On CSP for WASM:** `script-src 'self' 'wasm-unsafe-eval'` required. `wasm-unsafe-eval` is the modern CSP directive for instantiating WASM from same-origin without `unsafe-eval`. FR-A11Y-001 codifies this in the global CSP header.

**On Cowork Recipe support:** Decoder bump PRs touch this directory + 1-2 import lines. Easy review; Cowork Recipe A surfaces size delta in PR comment automatically.
