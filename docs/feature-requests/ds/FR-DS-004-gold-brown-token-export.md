## §1 — Description (BCP-14 normative)

The `@cyberskill/ds-cinematic/tokens/colors` module **MUST** export gold + brown ramps as both TypeScript const objects AND CSS custom properties, derived from `palette-canonical.json` (FR-DS-002).

1. **MUST** export a `gold` const object with keys `50, 100, 200, 400, 500, 600` mapped to hex strings byte-identical to `palette-canonical.json`.
2. **MUST** export a `brown` const object with keys `50, 100, 200, 400, 500, 700`.
3. **MUST** ship a `colors.css` companion file declaring `:root` CSS custom properties: `--brand-gold-50`, …, `--brand-gold-600`, `--brand-brown-50`, …, `--brand-brown-700`. Hex values MUST match the TS objects byte-identically.
4. **MUST** be typed as `Readonly<Record<string, `#${string}`>>` — TypeScript catches typos at consumer-import time.
5. **MUST NOT** introduce any colour outside FR-DS-002's locked 12 swatches (6 gold + 6 brown). Adding a colour requires an FR-DS-NNN amendment.
6. **MUST** be generated by a one-shot script `packages/ds-cinematic/scripts/gen-color-tokens.mjs` that reads `design/tokens/palette-canonical.json` and emits both `colors.ts` and `colors.css`. The generated files include a header `// GENERATED — re-run gen-color-tokens.mjs after editing palette-canonical.json`.
7. **MUST** ship a Vitest unit test asserting every TS export hex matches the canonical JSON byte-for-byte.

---

## §2 — Why this design (rationale for humans)

**Why generated, not hand-typed?** Hand-typed tokens drift against the canonical JSON. The script is the single transform; consumers import only from `colors.ts`/`colors.css`. If you edit palette-canonical.json without re-running the script, the test catches it.

**Why both TS const and CSS variables?** TS consumers want type-safety + tree-shaking (`colors.gold[400]` gets DCE'd if unused). CSS consumers (R3F shaders that read `getComputedStyle`, plain CSS files) want the `var(--brand-gold-400)` form. Both representations come from the same source.

### §3.3 `gen-color-tokens.mjs` shape

```js
#!/usr/bin/env node
import { readFileSync, writeFileSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const HERE = dirname(fileURLToPath(import.meta.url));
const palette = JSON.parse(readFileSync(join(HERE, '../../../design/tokens/palette-canonical.json'), 'utf8'));

function tsRamp(name, ramp) {
  const body = Object.entries(ramp)
    .map(([k, v]) => `  ${k.padEnd(3)}: '${v}',`).join('\n');
  return `export const ${name} = {\n${body}\n} as const satisfies Readonly<Record<string, \`#\${string}\`>>;`;
}
function cssRamp(prefix, ramp) {
  return Object.entries(ramp).map(([k, v]) => `  ${prefix}${k}: ${v};`).join('\n');
}

const ts = `// GENERATED — re-run gen-color-tokens.mjs after editing palette-canonical.json. Do NOT hand-edit.

${tsRamp('gold', palette.gold)}

${tsRamp('brown', palette.brown)}

export type GoldRamp = keyof typeof gold;
export type BrownRamp = keyof typeof brown;
`;
const css = `/* GENERATED — re-run gen-color-tokens.mjs after editing palette-canonical.json. */
:root {
${cssRamp('--brand-gold-', palette.gold)}

${cssRamp('--brand-brown-', palette.brown)}
}
`;
writeFileSync(join(HERE, '../src/tokens/colors.ts'), ts);
writeFileSync(join(HERE, '../src/tokens/colors.css'), css);
console.log('OK — wrote colors.ts + colors.css');
```

## §4 — Acceptance criteria

1. **colors.ts present + typed** — File exists; `pnpm -F @cyberskill/ds-cinematic typecheck` MUST exit 0.
2. **colors.css present** — File exists at the documented path.
3. **TS hex values match canonical JSON** — Vitest assertion: every value in `gold` + `brown` exports matches `palette-canonical.json` byte-for-byte.
4. **CSS hex values match canonical JSON** — Shell assertion: parse colors.css, extract every `--brand-*` value, diff against canonical JSON.
5. **GENERATED header present in both files** — `grep -c '^// GENERATED' colors.ts` >= 1; `grep -c '^/\* GENERATED' colors.css` >= 1.
6. **gen-color-tokens.mjs is idempotent** — Run twice, sha256 of outputs match.
7. **No off-palette colours** — `grep -E '#[0-9A-Fa-f]{6}' packages/ds-cinematic/src/tokens/colors.{ts,css}` MUST only return hex values in the canonical JSON.
8. **Re-generation test in CI** — `git diff --exit-code` after running the generator MUST be clean (catches stale generated files).

## §5 — Verification

```ts
// packages/ds-cinematic/src/tokens/__tests__/colors.test.ts
import { describe, expect, test } from 'vitest';
import palette from '../../../../../design/tokens/palette-canonical.json' assert { type: 'json' };
import { gold, brown } from '../colors';

describe('FR-DS-004 — color tokens', () => {
  test('AC#3: gold matches canonical', () => {
    expect(gold).toEqual(palette.gold);
  });
  test('AC#3: brown matches canonical', () => {
    expect(brown).toEqual(palette.brown);
  });
  test('AC#7: no off-palette', () => {
    const all = [...Object.values(gold), ...Object.values(brown)];
    const canonical = new Set([...Object.values(palette.gold), ...Object.values(palette.brown)]);
    for (const hex of all) expect(canonical.has(hex)).toBe(true);
  });
});
```
