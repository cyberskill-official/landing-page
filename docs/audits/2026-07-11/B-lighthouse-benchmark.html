Strong foundation, clear mobile fix

Executive summary

CyberSkill's site sits on an unusually strong technical base. In a mobile Lighthouse test run head-to-head against three Vietnamese agencies, it ships the lightest page of the group, 573 KB against 1.2 MB to 12.8 MB, and it scores a perfect 100 for both accessibility and best practices, which none of the three competitors reach. Its structured data is richer than most of them.

One problem holds the mobile result back. Lighthouse rates mobile performance at 47, while the same page scores 97 on desktop. The gap is not about page weight. It comes from a single mobile layout shift, a cumulative layout shift of 0.431 that is the worst in the benchmark, plus the cost of running roughly 900 KB of JavaScript on a throttled phone. Because the page is already lighter and its blocking time is already lower than the faster-scoring competitors, fixing the layout shift alone would move CyberSkill to the front of this group.

The remaining findings are smaller and mostly about reach rather than build quality. The sitemap lists only the two homepages, there are no social or authority links anywhere, a few links read "Learn more" with no context, and there is no llms.txt for AI answer engines. The plan repairs the mobile score first, then closes the search, trust, and content gaps.

The one-line verdict

Built well, under-marketed, and one layout fix away from leading its peer set on mobile.

CyberSkill wins page weight, accessibility, and best practices outright, and its blocking time is lower than both faster-scoring rivals. How this was measured

The live site was loaded in a real Chromium browser at mobile (390 px), tablet (768 px), and desktop (1440 px) widths. Performance, accessibility, best practices, and SEO were scored with Google Lighthouse, run with its standard mobile throttling and again in desktop mode, using the same settings for CyberSkill and for every competitor so the comparison is fair.

Accessibility was cross-checked with axe-core, the same engine Lighthouse uses. Page weight, compression, caching, security headers, redirects, structured data, sitemap, and robots rules were read directly from the server responses. The Vietnamese pages and the service, work, careers, and privacy routes were crawled to check content and metadata.

Honest caveats
  • Lab timings such as time to first byte pass through an audit proxy, so treat them as directional
  • Layout shift, blocking time, byte weight, and script totals are read from the page and are reliable
  • Lighthouse performance uses simulated throttling, so scores model a mid-range phone, not one specific device
  • Competitor scores are a single run each and can vary a few points between runs
